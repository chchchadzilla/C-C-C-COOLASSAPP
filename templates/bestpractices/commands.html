🖥️ CLI Launch Commands

claude

Start interactive session in current directory

claude "task description"

Start session with an initial prompt

claude -p "prompt"

Print mode — non-interactive, outputs result and exits

claude -p "prompt" --output-format json

JSON output for scripting and automation

claude --continue

Resume the most recent session

claude --resume <session-id>

Resume a specific past session

claude --model claude-sonnet-4-20250514

Use a specific model

claude --dangerously-skip-permissions

Full auto-accept mode (use with CI guardrails!)

cat file.ts | claude -p "review this"

Pipe input to Claude for processing

claude config set model claude-sonnet-4-20250514

Set default model globally

⚡ Interactive Slash Commands

CommandWhat It DoesWhen to Use
/compact Compresses conversation context, freeing up token space When agent gets slow, confused, or context-heavy. Use every 20-30 exchanges.
/clear Clears entire conversation history Starting a completely different task. Nuclear option.
/cost Shows token usage and estimated cost for this session Budget monitoring, especially with multiple parallel agents.
/doctor Checks Claude Code installation and configuration health Troubleshooting setup issues, verifying environment.
/help Shows all available commands Quick reference.
/init Creates a CLAUDE.md file for the current project First-time project setup. Claude analyzes codebase and generates context.
/model Switch model mid-session Switching from Opus (complex planning) to Sonnet (execution).
/permissions View and manage tool permissions Reviewing what the agent can do, adjusting allowlists.
/status Shows current session status, model, context usage Checking remaining context budget, current configuration.
/vim Toggle vim keybindings in the input If you're a vim user.

⚙️ Settings Configuration

Claude Code reads settings from multiple locations (higher priority overrides lower):

1 (highest) .claude/settings.json Project-level — committed to git, shared with team
2 .claude/settings.local.json Project-level, personal — gitignored
3 (lowest) ~/.claude/settings.json Global — your personal defaults across all projects
Complete .claude/settings.json example
{
  "permissions": {
    "allow": [
      "Read",
      "Write", 
      "Bash(npm test:*)",
      "Bash(npm run lint:*)",
      "Bash(npm run build)",
      "Bash(git add:*)",
      "Bash(git commit:*)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(npx tsc --noEmit)"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(git push:*)",
      "Bash(npm publish:*)",
      "Bash(curl:*)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "write_file",
        "command": "npx prettier --write $CLAUDE_FILE_PATH"
      }
    ]
  }
}

🧠 Model Selection Guide

Claude Opus 4

claude-opus-4-20250514
Best reasoning Complex architecture Multi-step planning Most expensive Slower

Use for: Orchestrator agents, architecture decisions, complex refactoring, debugging hard issues

Claude Sonnet 4

claude-sonnet-4-20250514
Best value Fast execution Great for routine tasks Default model

Use for: 80% of agent work — implementation, tests, docs, bug fixes, code review

Cost optimization tip: Use Opus for the orchestrator agent (1 instance), Sonnet for all worker agents (9 instances). The orchestrator plans; workers execute. This typically saves 40-60% vs all-Opus.
