🚨 Failure Modes & Recovery

When running 10 agents in parallel, failures aren't if — they're when. Here's every failure mode and how to handle it.

← All Best Practices

⚔️ 1. Merge Conflicts

What Happens

Two agents modify the same file or overlapping code. When you merge their branches, git can't automatically resolve the differences.

Why It Happens

  • Poor file ownership boundaries
  • Shared utility files modified by multiple agents
  • Agents "helpfully" refactoring imports or formatting outside their scope
  • Package.json/lock file changes from multiple agents

Prevention & Recovery

🛡️
Prevention: Define file ownership in CLAUDE.md. Use git worktrees. Add "Do NOT modify files outside your assigned directory" to agent instructions.
🔧
Recovery: NEVER let agents auto-resolve conflicts. Pull the conflicting branches, manually resolve, or use a dedicated "conflict resolver" agent with BOTH branches visible.
📊
Monitoring: Track conflict frequency. If one file/module has frequent conflicts, it needs to be refactored or assigned to a single agent.

🌫️ 2. Context Drift

What Happens

An agent's context window fills up with irrelevant information, causing it to "forget" earlier instructions or make increasingly poor decisions.

Why It Happens

  • Long-running sessions without /compact
  • Agent reads too many files, diluting its focus
  • Conversation goes on tangents
  • CLAUDE.md too long, consuming context budget

Prevention & Recovery

🛡️
Prevention: Keep tasks focused and scoped. Use /compact regularly. Keep CLAUDE.md concise. Start new sessions for new tasks instead of reusing old ones.
🔧
Recovery: If output quality drops, start a fresh session. Use claude --resume only when context is still relevant. Better to restart than to fight drift.
📊
Warning Signs: Agent repeats itself, ignores CLAUDE.md rules, makes changes it was told not to, or asks questions already answered.

🔄 3. Contradictory Implementations

What Happens

Two agents solve the same problem differently. Agent A creates a REST endpoint with Express, Agent B creates the same endpoint with Fastify. Both "work" but they're incompatible.

Why It Happens

  • CLAUDE.md doesn't specify tech choices explicitly enough
  • Agents interpret vague requirements differently
  • No "single source of truth" for architecture decisions
  • Overlapping task assignments

Prevention & Recovery

🛡️
Prevention: Architecture decisions go in CLAUDE.md immediately. "We use Express, not Fastify." "State management is Zustand, not Redux." Leave ZERO ambiguity.
🔧
Recovery: Pick the better implementation, discard the other. Update CLAUDE.md so it doesn't happen again. This is a documentation failure, not an agent failure.
📊
Detection: Code review agent can scan for conflicting patterns: multiple HTTP frameworks, duplicate implementations, incompatible interfaces.

💸 4. Resource Exhaustion

What Happens

Running 10 agents burns through API credits rapidly. An infinite loop in one agent can consume your daily budget in minutes.

Why It Happens

  • Agent stuck in retry loops
  • Reading very large files repeatedly
  • Full-auto mode with no guardrails
  • Not monitoring token usage per agent

Prevention & Recovery

🛡️
Prevention: Set per-agent token budgets. Monitor usage in this dashboard. Use claude-sonnet for routine tasks (cheaper), claude-opus only for complex work.
🔧
Recovery: Kill runaway agents immediately. Check the terminal output for loops. Add explicit "If this fails 3 times, stop and report the error" to CLAUDE.md.
📊
Budget Rule of Thumb: 10 active agents ≈ 1-2M tokens/hour with moderate tasks. Set alerts at 80% of your daily budget.

🏚️ 5. Cascading Failures

What Happens

Agent A breaks a shared module. Agents B, C, D all depend on it. They each try to "fix" it differently, creating 4 divergent versions of the same fix.

Why It Happens

  • Shared code modified without coordination
  • No integration testing between agent outputs
  • Full-auto mode allowing agents to "fix" problems they shouldn't

Prevention & Recovery

🛡️
Prevention: Mark shared modules as "protected" in CLAUDE.md. Require human approval for shared file changes. Run a dedicated integration agent that validates cross-agent compatibility.
🔧
Recovery: Pause all agents. Fix the shared module on main. Rebase all agent branches. Resume agents from clean state.

📋 Failure Response Quick Reference

FailureSeverityFirst ResponseTime to Fix
Merge Conflict Medium Manual resolve, update ownership rules 5-15 min
Context Drift Low New session, /compact, check CLAUDE.md 2-5 min
Contradictory Implementation High Pick winner, update CLAUDE.md, discard loser 15-30 min
Resource Exhaustion Critical Kill runaway agent immediately 1-2 min
Cascading Failure Critical Pause ALL agents, fix shared module, rebase 30-60 min
