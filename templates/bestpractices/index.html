⚡ Quick Tips

🎯 Give verification commands

Always tell agents HOW to verify their work: "After changes, run npm test and fix failures". Agents that verify produce dramatically better results.

🌿 One branch per agent

Never let two agents work on the same branch. Use git worktrees for zero-overhead isolation: git worktree add ../project-feature feature-branch

📏 Be specific, not vague

Bad: "Fix the auth system"
Good: "In src/auth/login.ts, add rate limiting that blocks after 5 failed attempts in 15 minutes. Add tests in tests/auth.test.ts."

🔍 Use subagents for big tasks

Ask Claude to break complex work into subtasks and use subagents. The orchestrator agent plans, subagents execute in parallel.

🛡️ Start with default permissions

Resist going to --dangerously-skip-permissions early. Build trust with each agent first. Use allowlists in .claude/settings.json.

📊 Check /compact regularly

When agents get slow or confused, use /compact to compress context. Long conversations drift — keep them focused.
