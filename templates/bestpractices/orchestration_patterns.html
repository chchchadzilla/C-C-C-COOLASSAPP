🧠 The Mental Model: Skills → Subagents → Plugins

Think of extending Claude Code capabilities in three tiers of complexity:

Skills (Easiest)

What: Markdown files in ~/.claude/skills/ or .claude/skills/ that teach Claude new patterns.

When: You want Claude to follow a specific coding pattern, use a particular library in a specific way, or follow a complex workflow.

# .claude/skills/api-endpoint.md
When creating API endpoints:
1. Use Express Router
2. Add Zod validation for request body
3. Use asyncHandler wrapper
4. Return standardized response format
5. Add OpenAPI comment annotations

Example:
```typescript
router.post('/users', asyncHandler(async (req, res) => {
  const body = CreateUserSchema.parse(req.body);
  const user = await userService.create(body);
  res.json({ success: true, data: user });
}));
```

Subagents (Medium)

What: Claude spawns child agents to handle subtasks in parallel. The main agent orchestrates.

When: Tasks can be decomposed into independent pieces. The orchestrator plans, subagents execute.

# Prompt that triggers subagent usage:
"Refactor all service files to use the new error 
handling pattern. There are 8 services that can be 
done independently — use subagents to parallelize."

# Claude will automatically:
# 1. List the 8 services
# 2. Spawn a subagent for each
# 3. Each subagent refactors one service
# 4. Main agent collects results

Hooks & MCP (Advanced)

What: Custom scripts that run at lifecycle events (pre-tool, post-tool, notification). MCP servers extend Claude with custom tools.

When: You need automated guardrails, custom integrations, or domain-specific capabilities.

# .claude/settings.json — Hook example
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "write_file",
      "command": "npm run lint --fix $CLAUDE_FILE_PATH"
    }]
  }
}

# Runs lint auto-fix after every file write

🏗️ Orchestration Patterns for 10 Agents

Pattern 1: The Feature Factory

Structure: 1 orchestrator agent + 8-9 worker agents

The orchestrator breaks down a large feature into independent tasks and distributes them. Workers execute in parallel on separate branches.

📋 Orchestrator reads requirements
→
📦 Breaks into 8-9 tasks
→
🔀 Assigns to workers (git worktrees)
→
✅ Workers execute & verify
→
🔄 Orchestrator reviews & merges
Best for: New feature development, module rewrites, test coverage sprints

Pattern 2: The Pipeline

Structure: Sequential agents with handoffs

Each agent handles one phase. Output of one is input to next. Like an assembly line.

🔍 Agent 1: Research & Design
→
🏗️ Agent 2: Implementation
→
🧪 Agent 3: Testing
→
👁️ Agent 4: Code Review
→
📝 Agent 5: Documentation
Best for: Quality-critical features, compliance-heavy work, unfamiliar codebases

Pattern 3: The Swarm

Structure: All agents work independently on different concerns

No orchestrator. Each agent has a clear domain. They never interact directly.

🔧 Agent 1-3: Features
🧪 Agent 4-5: Tests
📝 Agent 6: Docs
👁️ Agent 7: Reviews
🔒 Agent 8: Security
⚡ Agent 9-10: Bug fixes
Best for: Large codebases with clear module boundaries, maintenance work, sprint backlogs

🔐 Permission Strategies

How you set permissions directly impacts both safety and velocity:

🛡️ Default (Safest)

Agent asks before every file write and shell command.

Use for: New agents, critical codepaths, unfamiliar developers

claude

⚡ Allowlisted Tools

Auto-approve specific tools, ask for everything else.

Use for: Trusted agents doing routine work

// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Read", "Write",
      "Bash(npm test:*)",
      "Bash(npm run lint)",
      "Bash(git add:*)",
      "Bash(git commit:*)"
    ]
  }
}

🚀 Full Auto (Fastest, Riskiest)

Agent does everything without asking. Only use with CI guardrails.

Use for: Isolated tasks with test coverage and branch protection

claude --dangerously-skip-permissions
Requires: comprehensive test suite, branch protection, code review before merge

🪝 Hooks: Automated Guardrails

Hooks run custom commands at key lifecycle points. Essential for maintaining quality at scale:

.claude/settings.json — Production hooks
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "write_file",
        "command": "npx prettier --write $CLAUDE_FILE_PATH && npx eslint --fix $CLAUDE_FILE_PATH"
      }
    ],
    "PreToolUse": [
      {
        "matcher": "bash",
        "command": "echo 'Command: $CLAUDE_TOOL_INPUT' >> /tmp/claude-audit.log"
      }
    ],
    "Notification": [
      {
        "command": "terminal-notifier -title 'Claude' -message '$CLAUDE_NOTIFICATION'"
      }
    ]
  }
}
Hook types: PreToolUse (before any tool), PostToolUse (after any tool), Notification (when agent needs attention). Use matcher to filter which tools trigger the hook.
